# 细化

【前后端】
前端做交互和显示，后端管理存储数据和计算；
本地应用，而不是web应用。

【过程】
所有的过程的进行速度，和时间挂钩，
所有的过程，统一写到一个配置文件中，
每一个过程，需要选择一个员工作为负责人，

【人员】
至少是研建用管维5类角色，研1，建5，用1(不计分)，管1，维护(1)
每个角色按分得的任务计分，先检查预算还够不够，任务结束统一更新；
任务完成速度和工资线性关系；
按天计费，项目预算花完就停产离职/罢工。

【新建项目时，资源参数可调】
创建任务的时候，设置资源模板，自动配置各个阶段投产人数。
项目预算；
随项目进行，模拟其他开销：办公室租金、水电等等

- 如何实现手动自动两种逻辑？自动只负责填数，手动其实也就是修改，然后均由手动点击触发。

【每个阶段】
左边是xml图；
右边是当前过程，发生的事情；
执行当前任务的员工；


【研】
支持上传格式化SRS文档；
支持新建项目后，手动添加需求的界面；
调用大模型，对需求进行评价和打分，设置一个打分标准，给出修改意见；
直接跳到需求文档提交这一步；

【如何控制过程？】
未完成灰色，当前进行中蓝色，已完成是黑色。

【和效能指标是什么关系？】
要实现40个指标的计算，，就是在总览里面显示。
因为指标不是固定只在一个阶段更新的。
问题在于，模型是高度简化的，那就需要，将每个过程细分，然后有一个窗口，显示当前过程发生了什么，更新对应指标。。。

【看板】
前端指标更新的逻辑是什么？
过程是后端进行的，后端完成相应状态的更新，向前端推送效能指标。

【里程碑，输出】
项目界面，每一个阶段完成之后，输出里程碑文件到项目文件夹，（其实也都是提前写好，预制在项目里的），这里输出到项目文件夹的意思是，外部可以看到，并且在前端提供打包下载的功能。

【登录界面】
提供不同角色，管理才能看到概览；
其他角色，没有概览；

【后端分为几个模块？】
- [ ] 不同项目可复用：接口模块、各个阶段的进程模拟、员工模块、还有什么模块？
需要从后端的运行逻辑入手：

【完成业务流程】
前端负责登录界面，不同角色登录，显示不同的模块组合。管理者界面有：项目列表，员工列表，看到当前所有项目，可以点击下载项目所有里程碑输出，可以看到所有员工信息；管理者才可以创建项目，创建时可以选择不同的资源模板，也可以进行修改，然后点击创建项目；
员工列表：只能看到当前自己负责的项目界面；
只提供管理者、员工两类角色。

管理者界面点击具体项目，也进入到具体的项目界面；
那么这里也就需要，所有项目的数据，由后端维护，前端界面点击进入项目界面，是通过向后端请求数据实现的。

进入到项目界面，第一次进入的时候，就是默认开始当前项目，之后每次进入，都是向后端请求当前任务状态；后端需要实现数据持久化保存，也就是要有一个项目的数据结构，用来保存项目的所有状态信息。

项目界面有：总览、看板、项目进度、里程碑，这几个标签页构成，可以点击跳转到不同界面；
总览显示当前项目资源、项目进度、几个核心效能指标；
看板显示项目所有的效能指标；
项目进度，左边是XML格式的时序图，右上是当前任务员工，右下是当前任务进度、当前任务细化的事件(需要自定义)，例如，编码实现这个阶段的事件就可以有：代码修改、代码提交、分支合并、运行失败，代码修改、运行成功，等等，每一个事件，都更新项目维护的数据；
里程碑，每一个阶段结束，都自动向这个项目的文件存储区，添加一个对应文件，代表阶段输出；

对于项目，需要选择40个效能指标，然后设计这些指标需要的具体事件，这些指标是在项目看板界面显示的，需要在后端维护这所有指标需要的数据结构，根据事件更新对应数据，然后向前端发送通知，更新对应效能指标数据；

【使用阶段】
当项目进度走到软件部署这一步，提供链接，打开一个模拟前端应用界面，这个是一个模拟公司监控系统的前端平台，模拟接入了公司非常多的传感器，在这个前端显示数据。

【意外事件模拟】
设计几个意外事件：新增需求、使用中发现bug，触发对应持续改进与迭代阶段。

【多进程支持】
支持多线程，多项目同时运行，每创建一个项目，就新建一个线程，创建当前项目的文件夹，用来维护这个项目所有的数据；
