# 某"研建用管维"演示验证系统总体设计说明书

## 1. 项目概述

### 1.1 项目背景与目标
本项目旨在研发某"研建用管维"演示验证系统，通过模拟完整的软件开发生命周期，从需求研发到建设、使用、管理和维护的全流程，为装备全生命周期管理提供有效的演示验证平台。

**核心价值**：
- 为甲方提供直观的"研建用管维"全流程演示
- 通过效能指标可视化展示各阶段的关键数据
- 支持不同角色的参与体验，模拟真实的工作场景
- 为装备全生命周期管理提供决策支持

### 1.2 系统性质说明
**重要说明**：这是一个**模拟软件开发过程**的应用系统，不是真实的开发工具：

- **模拟性质**：所有开发过程都是模拟执行，不产生真实的代码、文档或产品
- **预定流程**：所有事件都是按预定流程触发，模拟真实开发中的各种情况
- **模拟数据**：所有效能指标都是基于模拟数据计算，用于演示和教学目的
- **应用场景**：主要用于过程演示、效能分析、培训教学和项目管理研究

### 1.3 设计调整要点
根据设计调整文档要求，本系统将采用以下关键设计调整：
- **技术架构调整**：采用Electron桌面应用架构，通过.exe安装包部署，而非Docker容器化部署
- **应用模拟设计**：用阶段模块作为独立应用，模拟开发应用的使用过程，纯前端实现
- **前后端分离**：明确前端界面功能和后端模块职责，完整梳理业务流程

## 2. 系统功能设计

### 2.1 核心功能概述

#### 2.1.1 效能指标监控系统
**功能描述**：基于50个精选效能指标，实时监控"研建用管维"全流程的关键数据

**核心特性**：
- **5个动态指标**：实时更新，反映当前阶段的核心活动
- **45个静态指标**：过程结束时更新，提供阶段性的汇总数据
- **多维度展示**：趋势图、排行榜、分布图、概览卡片、进度条等多种可视化方式
- **智能告警**：指标异常时自动提醒，支持阈值设置

**业务价值**：
- 为管理者提供项目健康度的直观判断
- 帮助识别项目中的瓶颈和风险点
- 支持基于数据的决策制定

#### 2.1.2 过程管理引擎
**功能描述**：基于"研建用管维"五个阶段的18个核心过程，提供完整的过程管理和演示

**核心特性**：
- **阶段化流程**：严格按照研→建→用→管→维的顺序执行
- **角色协作**：不同角色在不同阶段承担不同职责
- **执行模式**：支持手动、自动、混合三种执行模式
- **事件驱动**：基于时序图的关键事件触发机制

**业务价值**：
- 确保项目按照标准流程执行
- 提供可重复的演示体验
- 支持过程优化和改进

#### 2.1.3 角色权限管理
**功能描述**：基于不同角色的职责和权限，提供差异化的系统访问体验

**角色定义**：
- **管理者**：全系统访问权限，项目创建、员工管理、系统配置
- **研发人员**：研阶段相关功能，需求分析、技术预研
- **建设人员**：建阶段相关功能，系统设计、编码实现
- **使用人员**：用阶段相关功能，系统使用、用户培训
- **管理人员**：管阶段相关功能，进度管理、团队协作
- **维护人员**：维阶段相关功能，问题诊断、系统维护

**业务价值**：
- 确保不同角色只能访问其职责范围内的功能
- 提供个性化的用户体验
- 支持角色间的协作和交接

### 2.2 十一大核心机制

十一大机制作为系统的核心功能增强，贯穿整个"研建用管维"流程：

**项目初始化机制**：
- **需求统一受理机制**：自动解析SRS文档，提取需求并生成项目任务
- **大模型智能评判机制**：AI评估需求质量，提供配置建议
- **资源模板管理机制**：标准化资源配置，支持不同项目类型

**项目执行控制机制**：
- **响应按责反馈机制**：每个过程都有对应的执行者，实时监控各环节状态，异常时自动标红
- **生态建设激励机制**：每个过程都有对应执行者，每完成一项任务，角色对应计分
- **时间倍率控制机制**：每个过程定义明确的工时，员工完成任务速度与员工工资呈线性关系
- **里程碑管理机制**：每个阶段完成时模拟生成里程碑文件，保存到项目数据区，所有文件都是预设模板
- **信息实时同步机制**：随着项目变化，所有指标实时更新

**项目协作管理机制**：
- **成果共享共用机制**：管理者可在项目界面下载每个项目的所有里程碑文件

**系统基础支撑机制**：
- **权限分级管控机制**：不同角色登录不同界面，确保访问相应功能
- **项目持久化存储机制**：退出程序时保存项目进度，重启时加载工作区数据继续运行

### 2.3 业务流程设计

#### 2.3.1 项目全生命周期流程

**项目启动阶段**：
1. 管理者创建新项目，选择资源模板（资源模板管理机制）
2. 上传SRS需求文档（需求统一受理机制）
3. 系统自动解析需求并生成项目任务
4. 大模型智能评判需求质量（大模型智能评判机制）
5. 系统自动分配团队成员和预算
6. 初始化项目数据和效能指标
7. 启动项目执行流程

**研阶段流程**：
1. 市场调研与需求收集（核心动态过程）
2. 技术可行性研究
3. 竞品分析与概念验证
4. 需求文档提交和确认
5. 阶段结束自动生成里程碑（里程碑管理机制）

**建阶段流程**：
1. 系统架构设计
2. 详细设计与UI/UX设计
3. 编码实现（核心动态过程）
4. 测试与质量保证
5. 构建与持续集成
6. 阶段结束自动生成里程碑

**用阶段流程**：
1. 软件部署与发布（核心动态过程）
2. 用户培训与推广
3. 用户反馈收集
4. 阶段结束自动生成里程碑

**管阶段流程**（贯穿全周期）：
1. 项目进度管理
2. 团队协作管理（核心动态过程）
3. 资源配置管理
4. 阶段结束自动生成里程碑

**维阶段流程**：
1. 问题诊断与修复（核心动态过程）
2. 性能优化与功能增强
3. 持续改进与迭代管理
4. 阶段结束自动生成里程碑

**全程支撑机制**：
- **响应按责反馈机制**：各阶段负责人实时监控，异常时自动标红
- **时间倍率控制机制**：根据演示需要调整各阶段执行速度
- **成果共享共用机制**：各阶段成果自动收集和共享
- **生态建设激励机制**：实时计算和展示成员贡献积分
- **权限分级管控机制**：确保不同角色访问相应功能
- **信息实时同步机制**：保证所有用户信息同步更新

#### 2.2.2 效能指标更新机制

**动态指标更新**（5个核心指标）：
- **新增需求趋势**：研阶段需求创建时实时更新
- **代码提交趋势**：建阶段代码提交时实时更新
- **运行按流水线排名**：用阶段部署执行时实时更新
- **工时排名**：管阶段工时记录时实时更新
- **新增缺陷趋势**：维阶段缺陷报告时实时更新

**静态指标更新**（45个其他指标）：
- **过程结束时更新**：每个过程完成时基于预设规则生成数值
- **阶段结束时更新**：每个阶段完成时生成汇总数据
- **项目结束时更新**：整个项目完成时生成最终数据

#### 2.2.3 执行模式设计

**手动模式**：
- 项目创建时使用手动输入资源配置
- 每个环节结束后手动触发进入下一个流程
- 用户完全控制模拟过程的节奏
- 适合详细演示和教学场景

**自动模式**：
- 项目创建时使用预设模板进行资源配置
- 每个环节结束后自动进入下一个预定流程
- 系统按照预设节奏自动推进模拟过程
- 适合快速演示和概览场景

**混合模式**：
- 项目创建时可以选择模板或手动输入
- 关键节点需要用户确认，其他环节自动推进
- 平衡效率和控制性
- 适合大多数演示场景

### 2.3 用户体验设计

#### 2.3.1 管理者体验
**登录后界面**：
- 项目总览：所有项目的状态、进度、关键指标
- 员工管理：团队成员信息、任务分配、绩效统计
- 系统设置：资源配置、流程配置、指标配置

**项目操作**：
- 创建项目：选择模板、配置参数、分配团队
- 监控项目：实时查看项目进度和指标变化
- 管理项目：暂停、恢复、删除项目

#### 2.3.2 员工体验
**登录后界面**：
- 个人工作台：当前任务、待办事项、个人绩效
- 项目访问：只能查看和操作分配的项目
- 任务执行：按照角色职责执行相应任务

**工作流程**：
- 接收任务：系统自动分配或管理者手动分配
- 执行任务：按照流程要求完成相应工作
- 提交成果：完成任务后提交相关文档或数据

#### 2.3.3 演示体验
**流程演示**：
- 时序图可视化：显示当前执行到哪个过程
- 事件实时显示：右下角显示当前发生的事件
- 指标实时更新：相关指标随事件发生实时变化

**交互控制**：
- 播放控制：开始、暂停、继续、重置演示
- 速度调节：调整演示速度，适应不同场景
- 节点跳转：可以跳转到特定阶段或过程

## 4. 关键业务场景

### 4.1 项目创建场景
**触发条件**：管理者需要创建新项目进行演示

**业务流程**：
1. 管理者登录系统，进入项目管理界面
2. 点击"创建项目"，选择资源模板（资源模板管理机制）
3. 上传SRS需求文档（需求统一受理机制）
4. 系统自动解析需求，提取需求条目和优先级
5. 大模型智能评判需求质量（大模型智能评判机制）
6. 根据评判结果调整项目配置
7. 系统自动分配团队成员和预算
8. 初始化项目数据和效能指标
9. 项目创建完成，进入待启动状态

**关键输出**：
- 项目实例创建成功
- 需求文档解析完成
- 智能评判结果生成
- 团队成员自动分配
- 效能指标初始化完成
- 项目文件夹创建

### 4.2 项目执行场景
**触发条件**：项目启动后，按照"研建用管维"流程执行

**业务流程**：
1. 系统按照时序图顺序执行18个核心过程
2. 每个过程执行时，更新相关的效能指标
3. 5个关键过程执行时，触发动态事件和指标更新
4. 响应按责反馈机制实时监控各环节状态
5. 时间倍率控制机制调整执行速度
6. 前端界面实时显示当前过程和事件信息
7. 过程完成后，生成相应的里程碑文档（里程碑管理机制）
8. 成果自动收集到共享库（成果共享共用机制）
9. 成员贡献积分实时更新（生态建设激励机制）

**关键输出**：
- 过程执行状态更新
- 效能指标实时更新
- 里程碑文档生成
- 项目进度推进
- 成果自动收集
- 积分排名更新

### 4.3 效能监控场景
**触发条件**：项目执行过程中，需要监控关键指标

**业务流程**：
1. 系统实时计算5个动态指标
2. 过程结束时更新45个静态指标
3. 信息实时同步机制推送指标变化
4. 前端界面以不同方式展示各类指标
5. 指标异常时触发告警机制
6. 支持指标历史数据查询和趋势分析
7. 权限分级管控机制控制不同角色的查看权限

**关键输出**：
- 指标数据实时更新
- 可视化图表展示
- 异常告警通知
- 历史数据分析
- 权限控制验证

### 4.4 角色协作场景
**触发条件**：不同角色需要协作完成项目任务

**业务流程**：
1. 各角色按照职责分工执行相应任务
2. 权限分级管控机制确保角色只能访问相应功能
3. 任务完成后，触发下一个角色的任务
4. 响应按责反馈机制监控任务执行状态
5. 系统记录各角色的工作时间和成果
6. 生态建设激励机制计算贡献积分
7. 支持角色间的任务交接和协作
8. 生成角色绩效报告

**关键输出**：
- 任务分配和完成状态
- 角色工作记录
- 协作效率统计
- 绩效评估报告
- 贡献积分更新

### 4.5 异常处理场景
**触发条件**：项目执行过程中出现异常情况

**业务流程**：
1. 响应按责反馈机制检测到异常
2. 系统自动标红显示异常状态
3. 信息实时同步机制推送异常通知
4. 向相关负责人发送处理提醒
5. 负责人处理并提交反馈
6. 系统更新状态并记录处理结果
7. 根据处理结果调整后续流程
8. 更新相关效能指标

**关键输出**：
- 异常状态标红显示
- 异常通知推送
- 处理反馈记录
- 状态更新确认
- 流程调整建议

### 5.1 业务价值
**演示价值**：
- 为甲方提供直观的"研建用管维"全流程演示
- 通过可视化展示各阶段的关键数据和指标
- 支持不同角色的参与体验，增强演示效果
- 十大机制提供丰富的交互体验和真实感

**培训价值**：
- 为团队成员提供标准化的流程培训
- 通过模拟实践提升团队协作能力
- 支持不同角色的专业化培训
- 十大机制提供完整的项目管理培训体系

**决策支持**：
- 基于效能指标提供数据驱动的决策支持
- 帮助识别项目中的瓶颈和风险点
- 支持过程优化和改进
- 十大机制提供智能化的决策辅助

**管理价值**：
- 十大机制提供完整的项目管理功能增强
- 智能化、自动化、游戏化的管理体验
- 标准化的流程管控和质量保证
- 实时的协作和知识共享平台

### 5.2 技术优势
**架构优势**：
- Electron桌面应用，部署简单，用户体验好
- 前后端分离，职责清晰，易于维护
- 模块化设计，支持功能扩展
- 十大机制采用微服务架构，易于独立开发和部署

**性能优势**：
- 只有5个指标需要实时计算，性能优异
- 大部分指标使用预设值，响应快速
- 本地数据库，数据访问效率高
- 十大机制采用异步处理，提升系统响应速度
