# 前端界面设计详细说明

## 1. 界面层次结构

### 1.1 应用整体结构
```
Electron 应用
├── 登录界面 (Login)
├── 管理者主界面 (AdminDashboard)
│   ├── 项目管理 (ProjectManagement)
│   ├── 员工管理 (EmployeeManagement)
│   └── 系统设置 (SystemSettings)
├── 员工工作界面 (EmployeeDashboard)
│   ├── 个人工作台 (PersonalWorkspace)
│   └── 项目访问 (ProjectAccess)
└── 项目详情界面 (ProjectDetail)
    ├── 总览 (Overview)
    ├── 看板 (Dashboard) - 50个指标展示 + 完整时序图
    ├── 项目进度 (Progress)
    ├── 里程碑 (Milestones)
    └── 五阶段模块 (UnifiedTimeline) - 统一时序图界面
        ├── 完整时序图（左侧）
        ├── 当前过程详情（中央）
        └── 事件历史（右侧）
```

## 2. 详细界面设计

### 2.1 登录界面 (Login.vue)

**布局设计**：
- 居中的登录卡片，背景为系统主题色渐变
- 左侧显示系统Logo和标题
- 右侧为登录表单区域

**界面元素**：
- 系统Logo和标题显示区域
- 用户名输入框（带用户图标）
- 密码输入框（带锁定图标）
- 角色选择下拉框（包含管理者、研发人员、建设人员、使用人员、管理人员、维护人员）
- 登录按钮（带加载状态）

**功能特性**：
- 表单验证（用户名、密码必填，角色必选）
- 记住登录状态
- 错误提示和加载状态
- 响应式设计，适配不同屏幕尺寸

### 2.2 管理者主界面 (AdminDashboard.vue)

**布局设计**：
- 顶部导航栏：系统标题、用户信息、退出按钮
- 左侧菜单栏：项目管理、员工管理、系统设置
- 主内容区域：根据菜单切换显示不同模块

**界面结构**：
- 顶部导航栏：系统标题、用户信息、退出按钮
- 左侧菜单栏：项目管理、员工管理、系统设置
- 主内容区域：根据菜单切换显示不同模块

#### 2.2.1 项目管理模块 (ProjectManagement.vue)

**功能区域**：
1. **操作工具栏**：
   - 新建项目按钮
   - 搜索框（按项目名称搜索）
   - 状态筛选器（全部/进行中/已完成/暂停）
   - 视图切换（卡片视图/列表视图）

2. **项目列表区域**：
   - 项目卡片展示（默认视图）
   - 项目信息：名称、状态、进度、预算使用情况
   - 操作按钮：查看详情、下载里程碑、暂停/继续、删除

**项目卡片设计**：
- 卡片头部：项目名称和状态标签
- 项目信息区域：
  - 当前阶段显示
  - 项目进度条
  - 预算使用情况（已用/总额及百分比）
  - 团队人数统计
- 卡片底部操作区：
  - 查看详情按钮
  - 下载里程碑按钮
  - 更多操作下拉菜单（暂停项目、删除项目）

#### 2.2.2 新建项目对话框 (CreateProjectDialog.vue)

**步骤化创建流程**：
1. **步骤1：基本信息**
   - 项目名称输入框
   - 项目描述文本域
   - 项目预算数字输入框（范围：1万-100万，步长1000）

2. **步骤2：资源配置**
   - 资源模板选择（单选按钮组）
   - 团队配置表格：
     - 角色列（研、建、用、管、维）
     - 人数列（可调整，范围0-10）
     - 日薪列（显示元/天）
     - 预估成本列（自动计算月成本）
   - 时间倍率滑块（范围0.1-3倍，步长0.1）

3. **步骤3：确认创建**
   - 项目信息汇总展示
   - 包含：项目名称、预算、团队人数、月成本、时间倍率、预估完成时间

4. **操作按钮**
   - 取消按钮
   - 上一步/下一步按钮（根据当前步骤显示）
   - 创建项目按钮（最后一步，带加载状态）

#### 2.2.3 员工管理模块 (EmployeeManagement.vue)

**界面布局**：
- 顶部工具栏：添加员工、批量操作、搜索筛选
- 员工列表表格：显示员工基本信息、当前状态、绩效评分
- 右侧详情面板：显示选中员工的详细信息和任务历史

**员工列表表格**：
- 工具栏区域：
  - 添加员工按钮
  - 搜索框（按员工姓名搜索）
  - 状态筛选下拉框（全部/工作中/空闲/离线）

- 表格列设计：
  - 选择列（多选功能）
  - 头像列（显示员工头像或姓名首字母）
  - 姓名列
  - 角色列（带颜色标签区分不同角色）
  - 状态列（带颜色标签显示当前状态）
  - 当前项目列
  - 总积分列
  - 工作效率列（星级评分显示）
  - 日薪列
  - 操作列（编辑、删除按钮）

- 交互功能：
  - 行点击显示员工详情
  - 多选支持批量操作
  - 实时状态更新

### 2.3 员工工作界面 (EmployeeDashboard.vue)

**界面特点**：
- 简化的界面设计，专注于个人工作内容
- 个人工作台显示当前任务和待办事项
- 只能访问分配给自己的项目

**个人工作台布局**：
- 欢迎区域：
  - 用户头像和信息展示
  - 个人统计卡片（总任务数、已完成、总积分）

- 工作内容区域：
  - 当前任务时间线：
    - 任务卡片展示
    - 任务状态标识（未完成/进行中/已完成）
    - 任务详情（项目名称、阶段、进度条）
    - 操作按钮（查看详情、完成任务）
  
  - 我的项目网格：
    - 项目卡片展示
    - 项目基本信息（名称、阶段、进度）
    - 点击进入项目详情

### 2.4 项目详情界面 (ProjectDetail.vue)

#### 2.4.1 总览标签页 (ProjectOverview.vue)

**项目信息卡片**：

**功能设计**：
- 40个效能指标的分类展示

**界面布局**：
- 顶部控制区域：
  - 指标分类筛选下拉框（全部指标/质量指标/进度指标/成本指标/风险指标）
  - 刷新数据按钮（带加载状态）

  - 每个指标卡片包含：

#### 2.4.3 项目进度标签页 (ProjectProgress.vue)

**三栏布局设计**：
- 左侧：基于研建用管维角色交互的XML时序图可视化
- 右上：当前任务执行员工信息和状态
- 右下：任务进度和细化事件列表

**XML时序图设计逻辑**：
- 基于《研建用管维角色关系与效能指标体系分析.md》中的17个关键任务时序图
- 使用D3.js或SVG渲染五角色（研、建、用、管、维）的交互流程
- 显示任务执行顺序：需求分析→系统设计→部署应用→维护优化→持续改进
- 支持点击任务节点查看详细信息，右侧同步更新任务详情
- 颜色状态标识：未完成（灰色）、进行中（蓝色）、已完成（黑色）

**手动/自动执行模式控制**：
- 提供三种执行模式选择：手动模式、自动模式、混合模式
- 手动模式：所有任务进展需要用户手动点击触发
- 自动模式：系统自动填充任务数据和进度，但需要用户确认后才正式更新
- 混合模式：自动执行基础流程，允许用户随时手动干预和调整
- 自动模式下显示数据预览对话框，用户可选择确认、修改或重新生成

**界面布局**：
- 左侧时序图区域：
  - 时序图控制按钮（播放、暂停、重置）
  - 播放速度滑块（0.5-3倍速）
  - 基于研建用管维的XML时序图可视化
  - 阶段块显示（已完成/进行中/待开始状态）
  - 任务列表（任务状态、持续时间、主要角色）
  - 任务点击交互功能

- 右上当前任务执行人区域：
  - 执行人信息（头像、姓名、角色、状态）
  - 当前任务详情（名称、描述、进度条）
  - 无任务时的空状态提示

- 右下任务事件日志区域：
  - 执行模式选择（手动/自动/混合模式）
  - 执行当前任务按钮（非自动模式下显示）
  - 事件时间线展示
  - 事件内容（标题、描述、影响标签）

## 3. 五阶段功能模块详细设计

**设计原则**：
- 基于《研建用管维角色关系与效能指标体系分析.md》中的17个关键任务流程
- 每个阶段都包含详细的任务事件模板，支持自定义事件配置
- 集成意外事件模拟机制，包括需求变更、Bug发现等场景
- 支持手动/自动执行模式，提供灵活的项目控制方式

### 3.1 研阶段模块 (ResearchPhase.vue)

**功能设计逻辑**：
- **需求统一受理机制**：提供SRS文档上传界面，支持Word、PDF格式自动解析
- **大模型智能评价**：集成OpenAI等大模型API，对需求进行五维度评估
- **需求管理面板**：支持需求导入、分解、跟踪和变更管理
- **多方案对比工具**：提供参数化权衡模型，支持技术方案对比分析

**任务事件模板**：
- 需求分析任务包含：开始需求分析→需求文档撰写→需求评审准备→需求评审会议→需求修订
- 每个事件都有明确的持续时间和对效能指标的影响
- 支持手动触发和自动执行两种模式

**大模型评价逻辑**：
- 评分维度：清晰度、完整性、一致性、可行性、可测试性（各0-100分）
- 自动生成问题清单和改进建议
- 提供评估标准配置和结果可视化展示
- 支持评估结果导出和历史对比

### 3.2 用阶段模块（独立应用）

**特殊设计逻辑**：
- 用阶段作为独立的前端应用，纯前端实现，无需后端支持
- 模拟真实的监控系统使用场景，展示软件价值体现过程
- 当项目进度到达软件部署步骤时，提供链接打开此独立应用

**功能实现逻辑**：
- **实时数据监控**：模拟传感器数据生成（温度、湿度、压力、流量等）
- **阈值告警机制**：数据超过预设阈值时自动触发告警，发送到维护阶段
- **用户操作模拟**：模拟管理员日常巡检、参数调整、异常处理等操作
- **性能指标展示**：实时显示系统可用率、响应时间、用户满意度等指标

**意外事件触发**：
- 模拟使用过程中发现严重Bug，自动触发维护流程
- 支持手动触发各种异常场景，演示系统应对能力
- 用户反馈收集功能，为需求迭代提供输入
