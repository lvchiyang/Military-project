# 研建用管维演示验证系统 - 核心功能实现

## 1. 项目结构

```
military-project/
├── src/
│   ├── main/                    # Electron主进程
│   │   ├── index.ts            # 主进程入口
```

## 2. 核心功能模块

### 2.1 效能指标计算模块

基于50个效能指标的定义，实现指标计算和更新机制：

**核心特性**：
- 5个动态指标实时更新
- 45个静态指标过程结束时更新
- 支持手动/自动/混合执行模式
- 基于时序图流程的指标触发

**指标分类**：
- 趋势类指标（15个）：折线图展示
- 排名类指标（12个）：排行榜展示
- 分布类指标（10个）：饼图/柱状图展示
- 概览类指标（8个）：卡片/仪表盘展示
- 效率类指标（5个）：进度条/仪表盘展示

### 2.2 过程管理模块

基于角色交互时序图的18个核心过程：

**研阶段过程**：
1. 项目启动与规划
2. 市场调研与需求收集（核心动态过程）
3. 技术可行性研究
4. 竞品分析与概念验证

**建阶段过程**：
5. 系统架构设计
6. 详细设计与UI/UX设计
7. 编码实现（核心动态过程）
8. 测试与质量保证
9. 构建与持续集成

**用阶段过程**：
10. 软件部署与发布（核心动态过程）
11. 用户培训与推广
12. 用户反馈收集

**管阶段过程**：
13. 项目进度管理
14. 团队协作管理（核心动态过程）
15. 资源配置管理

**维阶段过程**：
16. 问题诊断与修复（核心动态过程）
17. 性能优化与功能增强
18. 持续改进与迭代管理

### 2.3 登录界面模块

实现不同角色的登录界面：

**管理者登录界面**：
- 完整的系统功能访问权限
- 项目管理、员工管理、系统设置
- 所有项目的查看和操作权限

**员工登录界面**：
- 简化的个人工作台界面
- 只能访问分配的项目
- 个人任务和绩效查看

**角色类型**：
- 管理者（admin）
- 研发人员（research）
- 建设人员（build）
- 使用人员（use）
- 管理人员（manage）
- 维护人员（maintain）

## 3. 技术实现要点

### 3.1 Electron架构
- 主进程：数据库管理、后端服务、文件处理
- 渲染进程：Vue 3前端界面、用户交互
- IPC通信：主进程与渲染进程数据交换

### 3.2 数据存储
- SQLite数据库：项目数据、员工信息、指标数据
- 文件系统：里程碑文档、配置文件
- 本地存储：用户设置、缓存数据

### 3.3 实时更新机制
- WebSocket风格通信：指标实时更新
- 事件驱动：基于过程触发指标更新
- 状态管理：Pinia管理前端状态

### 3.4 配置驱动
- 流程配置：JSON格式定义18个过程
- 指标配置：50个指标的详细定义
- 模板配置：项目模板和资源模板

## 4. 开发优先级

### 第一阶段：核心框架
1. Electron应用基础架构
2. 数据库设计和初始化
3. 基础的前后端通信

### 第二阶段：登录系统
1. 用户认证模块
2. 角色权限管理
3. 不同角色的界面路由

### 第三阶段：效能指标
1. 50个指标的定义和计算
2. 5个动态指标的实时更新
3. 指标数据的可视化展示

### 第四阶段：过程管理
1. 18个过程的流程控制
2. 手动/自动执行模式
3. 时序图可视化

### 第五阶段：界面完善
1. 管理者界面功能
2. 员工界面功能
3. 项目详情界面

## 5. 关键设计原则

### 5.1 模拟性质
- 所有开发过程都是模拟执行
- 不产生真实的代码、文档或产品
- 用于演示和教学目的

### 5.2 配置驱动
- 所有流程和指标通过配置文件定义
- 支持灵活的参数调整
- 易于扩展和维护

### 5.3 用户体验
- 直观的界面设计
- 清晰的流程展示
- 实时的数据更新

### 5.4 性能优化
- 只有5个指标需要实时计算
- 大部分指标使用预设值
- 响应快速，体验流畅
